# Event Stormingの補足メモ

## イベントの定義と命名
- イベントとは、ドメイン内で何か重要なことが起こったときに発生するもの
- 動詞の過去形で表現する
  - ProductPurchased（商品が注文された）
  - ProductShipped（商品が出荷された）
- イベントはビジネスの重要な振る舞いを表すため、イベントに注目するとドメインの本質的な側面を理解しやすい

## 「関係」文法（T字形ERの考え方）
- R(Event, Resource): R(受注, 顧客) など
- R(Event, Event): R(受注, 請求) など
- R(Resource, Resource): R(従業員, 部門) など

## リソースとイベントの区別方法
### 簡易的な方法
見つけたモデルがリソース（モノ）かイベント（コト）か迷った場合、モデル名に「○○する」という動詞句を追加する。
- 「配属する」は意味が通るので、配属はイベント
- 「部署する」は意味が通らないので、部署はリソース
- 「分類する」は意味が通るが、分類はリソース

### 厳密な方法
以下のいずれかを満たすものはイベントとみなす。
- タイムスタンプが採取できる
- 監査証跡が保管されている

例:
- 「分類」はどちらも満たさないため、リソースとなる
